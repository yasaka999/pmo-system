# 用户角色文档

## 角色概览

| 角色 | 描述 | 使用频率 | 技术能力 |
|------|------|----------|----------|
| **系统管理员** | 系统配置和用户管理 | 低 | 高 |
| **PMO 管理员** | 项目组合管理和监控 | 高 | 中 |
| **项目经理** | 单个项目交付管理 | 高 | 中 |
| **项目成员** | 项目任务执行 | 中 | 低 - 中 |
| **只读用户** | 项目信息查看 | 低 | 低 |

## 角色 1：系统管理员 (Admin)

### 基本信息
- **角色代码**: `admin`
- **技术能力**: 高
- **使用场景**: 系统维护和配置

### 背景
- IT 部门技术人员
- 熟悉系统架构和数据库
- 负责系统部署和维护

### 目标
- ✅ 系统稳定运行
- ✅ 用户管理
- ✅ 数据备份
- ✅ 权限配置

### 痛点
- ❌ 系统故障响应慢
- ❌ 用户问题处理繁琐
- ❌ 数据备份手动操作

### 核心功能使用
| 功能 | 使用频率 | 重要性 |
|------|----------|--------|
| 用户管理 | 低 | 高 |
| 系统配置 | 低 | 高 |
| 数据备份 | 中 | 高 |
| 日志查看 | 中 | 中 |

### 使用场景
1. **新用户入职**
   - 创建用户账号
   - 分配角色权限
   - 发送登录信息

2. **系统维护**
   - 定期数据备份
   - 系统升级
   - 性能监控

3. **问题处理**
   - 查看系统日志
   - 排查故障
   - 恢复数据

### 内容需求
- 系统部署文档
- 备份恢复手册
- 故障排查指南
- API 文档

## 用户旅程地图

### 项目经理典型一天

```
09:00  登录系统，查看项目概览
       └─> 检查是否有延期任务
       
10:00  更新项目计划
       └─> 调整任务优先级
       
14:00  处理问题登记
       └─> 分配问题给团队成员
       
16:00  查看风险矩阵
       └─> 识别新风险
       
17:00  填写项目周报
       └─> 总结本周进展
```

### PMO 管理员典型一周

```
周一   查看周末项目状态变化
       └─> 识别异常项目
       
周三   收集项目周报
       └─> 生成周报汇总
       
周五   准备周会材料
       └─> 导出项目组合报告
       
月末   月度报告生成
       └─> 分析预算执行
```
